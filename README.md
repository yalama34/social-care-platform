# Сервис помощи социально уязвимым категориям граждан

## Описание проекта

Веб-приложение для связи маломобильных граждан с волонтерами, готовыми оказывать помощь в повседневных бытовых делах.

### Социальная проблема

В современном обществе многие маломобильные люди испытывают проблемы с:
- Социальной изоляцией и одиночеством
- Выполнением повседневных задач (покупка продуктов, вынос мусора, уборка)
- Доступом к медицинской помощи (доставка лекарств)
- Мобильностью и передвижением

Наша платформа решает эти проблемы, создавая сообщество взаимопомощи, где волонтеры могут оказывать практическую помощь и эмоциональную поддержку нуждающимся.

## Технологии

### Backend
- **FastAPI** - высокопроизводительный веб-фреймворк для Python
- **SQLAlchemy** - ORM для работы с базой данных
- **PostgreSQL** - реляционная база данных
- **JWT** - аутентификация и авторизация
- **WebSocket** - realtime коммуникация для чата

### Frontend
- **React** - библиотека для создания пользовательского интерфейса
- **React Router** - маршрутизация
- **CSS3** - стилизация
- **Figma** - дизайн

### Инфраструктура
- **Docker** - контейнеризация приложений
- **Docker Compose** - оркестрация многоконтейнерных приложений

## Реализованная функциональность (MVP)

### Аутентификация
- Регистрация и авторизация по почте
- Верификация
- JWT-токены для сессий

### Ролевая система
- **Пользователи** - люди, нуждающиеся в помощи
- **Волонтеры** - люди, готовые оказывать помощь

### Управление заявками
- Создание заявок на различные виды услуг:
  - Доставка продуктов
  - Доставка лекарств
  - Вынос мусора
  - Уборка помещений
  - Помощь в передвижении
  - Общение и эмоциональная поддержка
  - Другие услуги
- Указание адреса, времени выполнения и комментариев
- Просмотр ленты активных заявок волонтерами
- Принятие заявок волонтерами
- Оценка выполненной работы
- Рейтинг волонтеров в профиле

### Система жалоб и наказаний
- Отправка жалобы в деталях заявки
- Анализ отправлнной жалобы с помощью LLM с окончательным вердиктом
- При достаточной уверенности модели пользователь получает наказание, если нарушил правила платформы
- Отправка жалоб на рассмотрение администрации при недосточной уверенности

### Чат и коммуникация
- WebSocket-based realtime чат между пользователем и волонтером
- История сообщений

### Профили пользователей
- Информация о пользователе
- Возможность добавления описания/биографии

## Запуск проекта

### Требования
- Docker и Docker Compose
- Node.js 16+ (для локального запуска frontend)
- Python 3.9+ (для локального запуска backend)

### Запуск с помощью Docker (рекомендуемый способ)

1. Клонируйте репозиторий:
```bash
git clone https://github.com/yalama34/social-care-platform.git
cd social-care-platform
```

2. Создайте файл `.env` в корне проекта с переменными окружения:
```env
# Database
DATABASE_DB=social_care
DATABASE_USER=postgres
DATABASE_PASSWORD=your_secure_password
DATABASE_PORT=5432
DATABASE_HOST=localhost

# Yandex GPT (для модерации)
YANDEX_FOLDER_ID=your_folder_id
YANDEX_API_KEY=your_api_key

# Email (SMTP)
SMTP_HOST=smtp.gmail.com
SMTP_PORT=587
SMTP_USER=your_email@gmail.com
SMTP_PASSWORD=your_app_password
SMTP_USE_TLS=true
SMTP_USE_SSL=false
```

3. Запустите все сервисы:
```bash
docker-compose up --build
```

4. Приложение будет доступно по адресам:
- Frontend: http://localhost:3000
- Auth Service: http://localhost:8000
- Core Service: http://localhost:8001
- PostgreSQL: localhost:5432

### Локальный запуск

#### Backend сервисы

1. Установите зависимости для auth_service:
```bash
cd auth_service
pip install -r requirements.txt
```

2. Установите зависимости для core_service:
```bash
cd core_service
pip install -r requirements.txt
```

3. Запустите PostgreSQL (локально или в Docker):
```bash
docker run -d --name postgres -e POSTGRES_DB=social_care -e POSTGRES_USER=postgres -e POSTGRES_PASSWORD=your_password -p 5432:5432 postgres:16.4
```

4. Запустите сервисы:
```bash
# Auth service
cd auth_service
uvicorn app.main:app --host 0.0.0.0 --port 8000 --reload 

# Core service (в новом терминале)
cd core_service
uvicorn app.main:app --host 0.0.0.0 --port 8001 --reload
```

#### Frontend

1. Установите зависимости:
```bash
cd frontend
npm install
```

2. Запустите приложение:
```bash
npm start
```
